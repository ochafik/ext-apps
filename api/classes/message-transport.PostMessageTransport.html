    Class PostMessageTransport

    JSON-RPC transport using window.postMessage for iframe↔parent communication.

    This transport enables bidirectional communication between MCP Apps running in iframes and their host applications using the browser's postMessage API. It implements the MCP SDK's Transport interface.

    Security

    The eventSource parameter is required and validates the message source window by checking event.source. For views, pass window.parent. For hosts, pass iframe.contentWindow to validate the iframe source.

    Usage

    View:

    const transport = new PostMessageTransport(window.parent, window.parent);
    await app.connect(transport);

    Host:

    const iframe = document.getElementById("app-iframe") as HTMLIFrameElement;
    const transport = new PostMessageTransport(
      iframe.contentWindow!,
      iframe.contentWindow!,
    );
    await bridge.connect(transport);

    See

    • App.connect for View usage
    • AppBridge.connect for Host usage
